    CMake 入门笔记

    Haruka 2021/7/30 LinuxCMake

    # 工程结构

    每一级的 CMakelist 将下一级添加到工程组织中，用如下语句添加之后每一级都会有一个 CMmakelist 文件。这是 CMake 组织工程的方式。

    ADD_SUBDIRECTORY(%subdirectory%)
    1

    # 编译参数

    介绍编译参数之前，要了解编译器。

    以 GCC 编译器为例，为两种语言设置编译标准，生成调试符号，设置警告等其他参数。

    set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS}-std=c++11 -g -wall -%")
set(CMAKE_C_FLAGS "${CMAKE_C_FLAGS} -std=c99 -02 -%")
    1
    2

    还可以区分 64 位和 32 位的编译模式。

    set(CMAKE_C_FLAGS "${CMAKE_C_FLAGS} -m64")
set(CMAKE_C_FLAGS "${CMAKE_C_FLAGS} -32")
    1
    2

    其他使用方法，如配置 Debug 和 Release 两种编译模式。可以参考 GCC 官网手册。

    # 创建工程

    # 创建名为 helloworld 的工程

    project( helloworld )
    1

    # 给工程添加源文件

    set(src
    ${CMAKE_CURRENT_SOURCE_DIR}/main.cpp
)
    1
    2
    3
    • 注意，这里 main.cpp 对应 CMAKE_CURRENT_SOURCE_DIR 这个目录，意味着 CMakelist 所在的目录，其中就有 main.cpp 文件，语句将它添加到工程当中。

    # 添加到可执行程序

    将 src(自取的变量名)对应的源文件添加到可执行程序里。

    add_executable( helloworld
    ${src}
    )
    1
    2
    3
    • 注意添加格式为${}。

    # 添加查找包含的目录

    引用工程以外的头文件需要手动将头文件的目录添加进来。

    include_directories(
    ${CMAKE_SOURCE_DIR}/samples/base64
    //以base64下工程为例
)
    1
    2
    3
    4
    • 注意，CMAKE_SOURCE_DIR 对应 CMake 工程的根目录，即工程第一级目录。这里去到/samples/base64路径下找到需要的头文件。

    # 设置库目录

    必须将头文件对应的实现以第三方库的形态（通常是静态库和动态链接库）添加进来，否则会输出undifined reference的错误。

    • 对于工程内部静态库，只需要将子级实现添加到当前工程。
    add_library ( base64 STASTIC
    ${src}
)
    1
    2
    3
    • 对于工程外部静态库，先添加查找库所在的目录，再将库的实现打上双引号，这是链接工程之外库的规则。
    link_directories(
    ${CMAKE_SOURCE_DIR}/build/samples/base64
)

target_link_libraries(
    linkage
    "base64" #以libbase64.a这个名字查找
    #base64_dynamic

    # 引用动态库

    CMake 中，在工程里设置好编译依赖后（完成了显式声明），在生成目标工程之前会给我们编译好需要的库。 动态库会生成两个文件，一个是动态库，一个是对应静态库，这个静态库其实是导入库，会拥有.a后缀，链接的时候会链接这个静态导入库，运行时加载的是动态库。所以运行时需要手动在工程目录加入这个动态库，否则运行会报错。

    # 链接编译器自带静态库

    只需要使用target_link_libraries()命令即可链接到工程，如链接库ws_32.a的语句为：

    target_link_libraries(
    ws_32
)

    2
    3

    不需要另外使用link_directories命令。
